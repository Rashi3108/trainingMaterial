Git
Question 1
Which Git command is used to create a new branch and switch to it in a single command?
  • git branch new-feature && git checkout new-feature
  • git checkout -b new-feature
  • git switch -c new-feature
  • Both B and C are correct
Correct Answer: D) Both B and C are correct
Both `git checkout -b new-feature` and `git switch -c new-feature` create a new branch and switch to it. The `git switch` command is newer and more intuitive for branch operations.
Git
Question 2
What is the difference between `git merge` and `git rebase`?
  • Merge creates a merge commit, rebase rewrites commit history
  • Merge is faster than rebase
  • Rebase creates a merge commit, merge rewrites history
  • There is no difference between them
Correct Answer: A) Merge creates a merge commit, rebase rewrites commit history
Git merge preserves the commit history and creates a merge commit, while git rebase rewrites the commit history by replaying commits on top of another branch, creating a linear history.
Git
Question 3
Which Git workflow strategy is best suited for continuous deployment environments?
  • Git Flow
  • GitHub Flow
  • GitLab Flow
  • Centralized Workflow
Correct Answer: B) GitHub Flow
GitHub Flow is ideal for continuous deployment as it's simple, has only one main branch, and supports frequent deployments. It's lightweight compared to Git Flow which has multiple long-lived branches.
Git
Question 4
What does `git stash` do?
  • Permanently deletes uncommitted changes
  • Temporarily saves uncommitted changes
  • Creates a new branch with current changes
  • Commits changes to the current branch
Correct Answer: B) Temporarily saves uncommitted changes
Git stash temporarily saves your uncommitted changes (both staged and unstaged) so you can work on something else, then come back and re-apply them later using `git stash pop` or `git stash apply`.
Jenkins
Question 5
What is the main advantage of Jenkins Pipeline jobs over Freestyle jobs?
  • Pipeline jobs are faster to execute
  • Pipeline jobs support version control of job configuration
  • Pipeline jobs use less system resources
  • Pipeline jobs don't require plugins
Correct Answer: B) Pipeline jobs support version control of job configuration
Pipeline jobs use Jenkinsfile which can be stored in version control, allowing job configuration to be versioned, reviewed, and shared. This provides better traceability and collaboration compared to GUI-based Freestyle jobs.
Jenkins
Question 6
In a Jenkins Declarative Pipeline, which section is used for cleanup activities regardless of build result?
  • stages
  • post { always }
  • post { success }
  • environment
Correct Answer: B) post { always }
The `post { always }` section in a Declarative Pipeline runs regardless of the build result (success, failure, or unstable). It's commonly used for cleanup activities like `cleanWs()` or sending notifications.
Jenkins
Question 7
Which Jenkins plugin is essential for integrating with GitHub webhooks?
  • Git Plugin
  • GitHub Integration Plugin
  • Pipeline Plugin
  • Maven Integration Plugin
Correct Answer: B) GitHub Integration Plugin
The GitHub Integration Plugin enables Jenkins to receive webhook notifications from GitHub, allowing automatic triggering of builds when code is pushed to the repository.
Jenkins
Question 8
What is the correct syntax to define tools in a Jenkins Declarative Pipeline?
  • tools { maven 'Maven-3.8.1'; jdk 'JDK-11' }
  • tools { maven: 'Maven-3.8.1', jdk: 'JDK-11' }
  • tools { maven 'Maven-3.8.1' jdk 'JDK-11' }
  • tools { maven('Maven-3.8.1'), jdk('JDK-11') }
Correct Answer: C) tools { maven 'Maven-3.8.1' jdk 'JDK-11' }
In Jenkins Declarative Pipeline, the tools section uses the syntax `toolType 'toolName'` with each tool on a separate line, without semicolons or commas.
SonarQube
Question 9
What are the three main quality dimensions that SonarQube measures?
  • Performance, Security, Scalability
  • Reliability, Security, Maintainability
  • Bugs, Vulnerabilities, Code Coverage
  • Complexity, Duplication, Documentation
Correct Answer: B) Reliability, Security, Maintainability
SonarQube measures code quality across three main dimensions: Reliability (bugs), Security (vulnerabilities), and Maintainability (code smells). Each dimension has its own rating from A (best) to E (worst).
SonarQube
Question 10
Which SonarQube component is responsible for analyzing source code locally?
  • SonarQube Server
  • SonarQube Database
  • SonarQube Scanner
  • SonarLint
Correct Answer: C) SonarQube Scanner
SonarQube Scanner analyzes the source code locally and sends the analysis results to the SonarQube Server. Different scanners exist for different build tools (Maven, Gradle, CLI).
SonarQube
Question 11
What is the purpose of Quality Gates in SonarQube?
  • To control user access to projects
  • To define quality criteria that must be met before deployment
  • To schedule automatic code analysis
  • To manage SonarQube plugins
Correct Answer: B) To define quality criteria that must be met before deployment
Quality Gates define a set of conditions (like coverage > 80%, no critical bugs) that code must pass before it can be deployed. They act as a checkpoint in CI/CD pipelines to ensure code quality standards.
SonarQube
Question 12
Which dependency is NOT required for running SonarQube?
  • Java 11 or 17
  • Elasticsearch (embedded)
  • Docker
  • Database (PostgreSQL recommended)
Correct Answer: C) Docker
Docker is not required to run SonarQube. While Docker simplifies deployment, SonarQube can run directly on a system with Java, embedded Elasticsearch, and a database. Docker is just one deployment option.

Question 14
Which command would you use in a Jenkins pipeline to run SonarQube analysis for a Maven project?
  • sh 'sonar-scanner'
  • sh 'mvn sonar:sonar'
  • sh 'mvn sonarqube:analyze'
  • sh 'mvn clean sonar'
Correct Answer: B) sh 'mvn sonar:sonar'
For Maven projects, `mvn sonar:sonar` is the correct command to run SonarQube analysis. This uses the SonarQube Maven plugin to analyze the code and send results to the SonarQube server.

Question 15
What is the benefit of using SonarLint in VS Code alongside SonarQube server analysis?
  • SonarLint replaces the need for SonarQube server
  • SonarLint provides real-time feedback while coding, SonarQube provides comprehensive project analysis
  • SonarLint is faster than SonarQube server analysis
  • SonarLint can analyze more programming languages
Correct Answer: B) SonarLint provides real-time feedback while coding, SonarQube provides comprehensive project analysis
SonarLint and SonarQube server complement each other: SonarLint gives immediate feedback as you write code in your IDE, while SonarQube server provides comprehensive analysis of the entire project, quality gates, and historical trends.
