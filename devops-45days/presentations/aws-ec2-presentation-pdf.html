Amazon EC2 - Elastic Compute Cloud

What is EC2?

Amazon Elastic Compute Cloud (EC2) provides scalable computing capacity in the Amazon Web Services (AWS) cloud. It eliminates the need to invest in hardware upfront, allowing you to develop and deploy applications faster.

Key Features

  • Virtual servers in the cloud
  • Complete control over the operating system
  • Flexible instance types for different workloads
  • Pay-per-use pricing model
  • Integration with other AWS services

Common Use Cases

  • Web applications and websites
  • Development and testing environments
  • Big data processing and analytics
  • Machine learning workloads
  • Enterprise applications

EC2 Key Concepts

Essential Components

Amazon Machine Images (AMI)

Pre-configured templates for your instances that contain the operating system, application server, and applications needed to launch your instance.

  • Amazon Linux 2: AWS-optimized Linux distribution
  • Ubuntu: Popular open-source Linux distribution
  • Windows Server: Microsoft Windows-based instances
  • Custom AMIs: Your own configured images

Security Groups

Virtual firewalls that control inbound and outbound traffic to your instances.

Common Security Group Rules:

  • HTTP (Port 80): Allow web traffic from anywhere (0.0.0.0/0)
  • HTTPS (Port 443): Allow secure web traffic
  • SSH (Port 22): Allow secure shell access for Linux
  • RDP (Port 3389): Allow remote desktop for Windows

Key Pairs

Secure login credentials consisting of a public and private key pair used for authentication.

  • Public Key: Stored on the EC2 instance
  • Private Key: Downloaded and stored securely by you
  • Usage: Required for SSH access to Linux instances

Security Note: Never share your private key files. Store them securely and set proper file permissions (chmod 400).

Method 1: User Data Automation

What is User Data?

User Data is a feature that allows you to pass data to your instance when it launches. This data can be used to perform common automated configuration tasks and run scripts after the instance starts.

User Data Script

#!/bin/bash
# Update the system packages
yum update -y

# Install Apache HTTP Server
yum install -y httpd

# Start Apache service
systemctl start httpd

# Enable Apache to start automatically on boot
systemctl enable httpd

# Create a simple HTML web page
cat > /var/www/html/index.html << 'EOF'
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My First AWS Web Server</title>
    <style>
        body { font-family: Arial, sans-serif; margin: 50px; }
        .container { background: white; padding: 30px; 
                    border-radius: 10px; text-align: center; }
        h1 { color: #232f3e; }
        p { color: #666; font-size: 18px; }
    </style>
</head>
<body>
    <div class="container">
        <h1>Welcome to My AWS Web Server!</h1>
        <p>This server was created automatically using User Data.</p>
        <p>Deployment completed successfully!</p>
    </div>
</body>
</html>
EOF

# Set proper ownership and permissions
chown apache:apache /var/www/html/index.html
chmod 644 /var/www/html/index.html

Key Benefits

  • Automation: No manual intervention required
  • Consistency: Same setup every time
  • Speed: Faster deployment process
  • Scalability: Easy to replicate across multiple instances

Method 2: Manual Setup

Step-by-Step Process

This method involves connecting to your EC2 instance via SSH and manually configuring the web server.

Step 1: Connect to Your Instance

# Connect via SSH (replace with your key and IP)
ssh -i "your-key.pem" ec2-user@your-instance-public-ip

# Example:
ssh -i "my-key.pem" ec2-user@54.123.45.67

Step 2: Update System and Install Apache

# Update the system packages
sudo yum update -y

# Install Apache HTTP Server
sudo yum install -y httpd

# Verify installation
httpd -version

Step 3: Start and Enable Apache

# Start Apache service
sudo systemctl start httpd

# Enable Apache to start on boot
sudo systemctl enable httpd

# Check Apache status
sudo systemctl status httpd

Step 4: Create Web Page

# Create the HTML file
sudo tee /var/www/html/index.html > /dev/null << 'EOF'
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>My Manual Web Server</title>
    <style>
        body { font-family: Arial, sans-serif; margin: 50px; }
        h1 { color: #232f3e; text-align: center; 
             border-bottom: 3px solid #ff9900; }
        .badge { background: #ff9900; color: white; 
                padding: 5px 15px; border-radius: 20px; }
    </style>
</head>
<body>
    <h1>Manually Configured Web Server!</h1>
    <div class="badge">Manually Configured</div>
    <p>This server was set up step by step manually.</p>
</body>
</html>
EOF

# Set proper permissions
sudo chown apache:apache /var/www/html/index.html
sudo chmod 644 /var/www/html/index.html

AWS CLI Commands

Essential EC2 Commands

Security Group Setup

# Create security group
aws ec2 create-security-group \
    --group-name web-server-sg \
    --description "Security group for web server demo" \
    --region us-east-1

# Allow HTTP traffic (port 80)
aws ec2 authorize-security-group-ingress \
    --group-name web-server-sg \
    --protocol tcp --port 80 --cidr 0.0.0.0/0 \
    --region us-east-1

# Allow SSH traffic (port 22)
aws ec2 authorize-security-group-ingress \
    --group-name web-server-sg \
    --protocol tcp --port 22 --cidr 0.0.0.0/0 \
    --region us-east-1

Instance Management

# Launch instance with user data
aws ec2 run-instances \
    --image-id ami-0abcdef1234567890 \
    --count 1 \
    --instance-type t2.micro \
    --key-name your-key-pair \
    --security-groups web-server-sg \
    --user-data file://userdata.sh \
    --region us-east-1

# List running instances
aws ec2 describe-instances \
    --query 'Reservations[*].Instances[*].[InstanceId,State.Name,PublicIpAddress]' \
    --output table --region us-east-1

# Stop instance
aws ec2 stop-instances --instance-ids i-1234567890abcdef0

# Start instance
aws ec2 start-instances --instance-ids i-1234567890abcdef0

# Terminate instance
aws ec2 terminate-instances --instance-ids i-1234567890abcdef0

Useful Queries

# Get instance public IP
aws ec2 describe-instances \
    --instance-ids i-1234567890abcdef0 \
    --query 'Reservations[0].Instances[0].PublicIpAddress' \
    --output text

# List available AMIs (Amazon Linux 2)
aws ec2 describe-images --owners amazon \
    --filters 'Name=name,Values=amzn2-ami-hvm-*' \
    --query 'Images[*].[ImageId,Name]' --output table

# List key pairs
aws ec2 describe-key-pairs --region us-east-1

Security Best Practices

Security Groups Configuration

üîí Inbound Rules

  • Principle of Least Privilege: Only open necessary ports
  • Specific Sources: Use specific IP ranges instead of 0.0.0.0/0 when possible
  • Regular Reviews: Periodically audit security group rules
  • Documentation: Document the purpose of each rule

üîë Key Management

  • Secure Storage: Store private keys securely
  • File Permissions: Set chmod 400 on key files
  • Key Rotation: Rotate keys regularly
  • No Sharing: Never share private keys

Common Security Group Rules

Service Port Protocol Source Purpose
HTTP 80 TCP 0.0.0.0/0 Web traffic
HTTPS 443 TCP 0.0.0.0/0 Secure web traffic
SSH 22 TCP Your IP/32 Remote access
RDP 3389 TCP Your IP/32 Windows remote desktop

Security Reminder: AWS follows a shared responsibility model. AWS secures the infrastructure, while you're responsible for securing your applications and data.

Additional Security Measures

  • IAM Roles: Use IAM roles instead of hardcoded credentials
  • VPC: Deploy instances in private subnets when possible
  • Monitoring: Enable CloudTrail and CloudWatch logging
  • Updates: Keep operating system and applications updated
  • Backup: Regular backups using EBS snapshots

Common SSH Issues

SSH Connection Problems

  • Wrong key file: Ensure you're using the correct .pem file
  • Key permissions: Key file must have 400 permissions
  • Wrong username: Use 'ec2-user' for Amazon Linux
  • Security group: Ensure port 22 is open
  • Public IP: Use the current public IP address

Web Server Issues

Apache HTTP Server Problems

  • Service not running: sudo systemctl start httpd
  • Port not accessible: Check security group rules
  • File permissions: Ensure proper ownership (apache:apache)
  • Configuration errors: Check /var/log/httpd/error_log
